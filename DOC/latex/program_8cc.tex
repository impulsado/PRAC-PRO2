\doxysubsection{Descripción detallada}
Programa principal de la pràctica. 

Aquest programa gestiona el comerç entre ciutats i un vaixell d\textquotesingle{}una cuenca fluvial. La cuenca és un arbre binari que representa les connexions entre les ciutats.

Aquest arbre només té els id\textquotesingle{}s de les ciutats, després és fa servir un conjunt de ciutats per obtenir les dades de les ciutats. Una ciutat pot tindre 2 ciutats fills o cap. De la estructura d\textquotesingle{}arbre tradicional, el root seria l\textquotesingle{}ultima ciutat de la cuenca mentre que les fulles serien les ciutats d\textquotesingle{}on comença un afluent.

Hi ha un conjunt de productes que no pot ser buit on es guarden els productes disponibles. El id comença per 1 i és incremental.

Una ciutat pot tindre o no inventari de productes. Aquest inventari es pot modificar afegint, eliminant o modificant productes.

Un vaixell sempre té un inventari vàlid on té una quantitat d\textquotesingle{}un determinat producte per a vendre i un altre per a comprar (diferents). Aquest inventari es pot modificar. El vaixell també té un registre de les ultimes ciutats on ha comerciat cada vegada que s\textquotesingle{}ha fet un viatge al llarg de la cuenca.

La classe \mbox{\hyperlink{classLlanxa}{Llanxa}} representa una versió del vaixell més reduïda que només s\textquotesingle{}utilitza per a calcular el millor viatge a realitzar. Això implica que només té la informació sobre els productes de compra i venta del vaixell juntament amb mètodes modificats per a ser més optims.

La classe viatge ajuda a organitzar les dades per a realitzar un viatge entre ciutats. Conté la quantitat total de productes comerciats si es realitza el viatge i les ciutats per on passa.

Les polítiques de programació són\+:
\begin{DoxyItemize}
\item Per a la classe \mbox{\hyperlink{classCjt__productes}{Cjt\+\_\+productes}} s\textquotesingle{}ha decidit fer servir un vector per a tenir un accés ràpid donat que el id és numèric i incremental.
\item Per a la classe \mbox{\hyperlink{classCjt__ciutats}{Cjt\+\_\+ciutats}} s\textquotesingle{}ha decidit fer servir un map per a tenir un accés més ràpid a les ciutats.
\item Per a l\textquotesingle{}operació de viatge s\textquotesingle{}ha decidit crear una nova classe per millorar la llegibilitat del codi.
\item No s\textquotesingle{}ha fet servir un Bin\+Tree de ciutats donat que modificar les ciutats dins de l\textquotesingle{}arbre gran perdem eficiencia.
\item Els paràmetres que es passen a una mètode de la classe sempre són correctes. 
\end{DoxyItemize}
